# Security Policy

## Supported Versions

Security updates are currently provided for the latest minor release line.

| Version | Supported |
|---|---|
| 0.3.x | Yes |
| 0.2.x and earlier | No |

## Reporting a Vulnerability

Please report security issues privately and do not open a public GitHub issue.

1. Use GitHub Security Advisories for this repository.
2. Include a clear impact statement, reproduction steps, and affected versions.
3. If possible, include a minimal proof of concept.

### What to expect

- Initial acknowledgement: within 72 hours
- Triage and severity assessment: within 7 days
- Fix timeline: depends on severity and exploitability

High-severity vulnerabilities may trigger an out-of-band patch release.
